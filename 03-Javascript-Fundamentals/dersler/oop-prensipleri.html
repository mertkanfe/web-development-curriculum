Ders 16: Nesnelerin Sırları

OOP'nin Süper Güçleri: Kapsülleme (Encapsulation) ve Kalıtım (Inheritance) 🛡️

1. Kapsülleme (Encapsulation) Nedir?

Bir önceki derste sınıflar (Class) oluşturup içlerine özellikler (örneğin hız, marka) eklemiştik. Ancak yazılım dünyasında verilerimizi ortalıkta bırakamayız. Onları koruma altına almamız gerekir. İşte verileri dışarıdan gelen tehlikelere karşı gizleme işlemine Kapsülleme diyoruz.

Kafamızda Canlandıralım: Televizyon ve Kumanda 📺
Televizyonun içindeki karmaşık kablolara, devrelere veya çiplere dokunamazsınız (onlar plastik bir kapsülün içindedir). Televizyonu kontrol etmek için size sadece izin verilen düğmeleri (Kumanda) kullanırsınız. Yazılımda da değişkenleri gizleriz (kablolar), onlara sadece özel fonksiyonlar (kumanda düğmeleri) aracılığıyla erişilmesine izin veririz.

2. Getter ve Setter Mantığı (Kumanda Düğmeleri)

Değişkenlerin başına private (özel) kelimesini koyduğumuzda, o değişkene dışarıdan kimse dokunamaz. Okumak için Get (Getir), değiştirmek için Set (Ayarla) metodlarını kullanırız.

class BankaHesabi {
    
    // private: Sadece bu sınıfın içinden erişilebilir (Gizli)
    private int bakiye = 1000;

    // GETTER: Bakiyeyi sadece okumaya izin veren fonksiyon
    public int getBakiye() {
        return bakiye;
    }

    // SETTER: Bakiyeye para eklerken kural koyduğumuz fonksiyon
    public void setBakiye(int miktar) {
        if (miktar > 0) {
            bakiye = bakiye + miktar;
            console.log("Para yatırıldı!");
        } else {
            console.log("Hata: Eksi değerde para yatıramazsınız!");
        }
    }
}
Neden Direk Değiştirmiyoruz?
Eğer bakiye gizli (private) olmasaydı, kötü niyetli biri dışarıdan gelip hesap.bakiye = -5000; yazarak sistemi çökertebilirdi. Kapsülleme sayesinde kuralları biz koyarız!

3. Kalıtım (Inheritance) Nedir?

Yazılımdaki en büyük kurallardan biri "Kendini Tekrar Etme" (DRY - Don't Repeat Yourself) kuralıdır. Eğer iki farklı sınıfın ortak özellikleri varsa, bunları tekrar tekrar yazmak yerine bir "Ata (Parent)" sınıf oluşturup özelliklerini "Çocuk (Child)" sınıflara miras bırakırız.

Kafamızda Canlandıralım: Hayvanlar Alemi 🦁
Kedi, Köpek ve Kuş farklı hayvanlardır ama hepsinin ortak özellikleri vardır (yemek yerler, uyurlar, yaşları vardır). Hepsine ayrı ayrı "yemekYe()" kodu yazmak yerine, bir Hayvan sınıfı oluştururuz. Kedi ve Köpek sınıfları, Hayvan sınıfından "Miras" alır.

4. Miras Alma İşlemi (Extends kelimesi)

Bir sınıfın başka bir sınıftan miras almasını istiyorsak, extends (genişletir) anahtar kelimesini kullanırız.

// 1. ATA SINIF (Miras Bırakan)
class Hayvan {
    String isim;
    
    void yemekYe() {
        console.log(isim + " şu an yemek yiyor.");
    }
}

// 2. ÇOCUK SINIF (Miras Alan)
// "Kedi sınıfı, Hayvan sınıfının tüm özelliklerini içine aldı!"
class Kedi extends Hayvan {
    
    void miyavla() {
        console.log("Miyav!");
    }
}

// KULLANIMI:
Kedi tekir = new Kedi();
tekir.isim = "Garfield";
tekir.miyavla(); // Kendi özelliği
tekir.yemekYe(); // Ata sınıftan gelen özellik!
Ne Kazandık?
Eğer oyuna bir "Köpek" eklemek istersek, yemek yeme veya uyuma kodlarını sıfırdan yazmayacağız. Sadece class Kopek extends Hayvan diyeceğiz ve saniyeler içinde tüm temel özelliklere sahip yepyeni bir canlı yaratmış olacağız!

OOP'nin Ustası Oluyorsunuz! 🎓

Kapsülleme ile verilerinizi güvence altına almayı (Güvenlik), Kalıtım ile de yazdığınız kodları tekrar tekrar kullanmayı (Verimlilik) öğrendiniz. Artık arka planda çalışan devasa sistemlerin nasıl bu kadar düzenli kalabildiğini biliyorsunuz!
