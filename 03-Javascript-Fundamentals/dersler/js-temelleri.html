Ders 2: JavaScript'e İlk Adım

Kodları Nereye Yazıyoruz, Çıktıları Nasıl Görüyoruz ve Değişkenler Nedir?

1. JavaScript Kodlarını Nereye Yazıyoruz?

Tıpkı CSS'te olduğu gibi, JavaScript kodlarımızı sitemize eklemenin birkaç yolu vardır. En profesyonel ve düzenli olanı, kodlarımızı harici (external) bir dosyada tutmaktır.

  • Harici Dosya (Önerilen): app.js adında bir dosya oluştururuz ve HTML dosyamızın en altına, </body> etiketinden hemen önce şu kodu ekleriz:
    <script src="app.js"></script>
Neden </body> etiketinden hemen önce?
Çünkü tarayıcılar kodları yukarıdan aşağıya doğru okur. Eğer JavaScript'i en üstte (Head içinde) yüklerseniz, HTML sayfadaki resimler ve butonlar henüz yüklenmediği için JavaScript hata verebilir. JS dosyası en son yüklenmelidir ki sahnedeki tüm oyuncular (HTML) yerini almış olsun!

2. Bilgisayarla Konuşma: Çıktı Alma Yöntemleri

Yazdığımız kodların çalışıp çalışmadığını görmek için sonuçları bir yere yazdırmamız gerekir. İşte en çok kullandığımız 3 yöntem:

1. console.log() 🕵️‍♂️

Yazılımcıların En İyi Dostu! Ekranda görünmez, sadece tarayıcının "Geliştirici Seçenekleri -> Konsol" sekmesinde görünür. Hataları bulmak için kullanırız.

// Sadece bizim görebileceğimiz gizli mesaj
console.log("Sistem başarıyla yüklendi!");

2. alert() 🔔

Sayfanın en üstünden aşağı doğru sarkan, kullanıcı "Tamam" diyene kadar gitmeyen o meşhur uyarı (pop-up) pencereleridir.

// Kullanıcının yüzüne çarpan mesaj
alert("Şifrenizi yanlış girdiniz!");

3. innerHTML 📝

HTML'in içine doğrudan müdahale etme yöntemidir. Sitedeki bir yazıyı (örneğin bir başlığı) anında değiştirmek için kullanırız.

// Ekrana yazı yazdırma
document.getElementById("baslik").innerHTML = "Yeni Başlık!";

3. Hafızayı Kullanmak: Değişkenler (Variables)

Programlama yaparken bazı bilgileri (kullanıcının adı, oyundaki skoru, bir ürünün fiyatı) hafızada tutmamız gerekir. İşte bu bilgileri sakladığımız alanlara Değişken diyoruz.

Kafamızda Canlandıralım: Etiketli Kutular 📦
Değişkenleri boş kutular olarak düşünün. Kutunun üzerine bir etiket yapıştırırsınız (İsim) ve içine bir eşya koyarsınız (Değer). İhtiyacınız olduğunda o kutuyu çağırır ve içindeki eşyayı kullanırsınız.

Modern JavaScript'te Kutuları Nasıl Tanımlarız?

Eskiden var kullanılırdı, ancak günümüzde modern ve daha güvenli olan let ve const kullanıyoruz.

  • let : Değişebilen Kutular
    Eğer kutunun içindeki değeri daha sonra değiştirecekseniz let kullanırsınız. Örneğin bir oyundaki puanınız sürekli değişir.
    let oyuncuSkoru = 0; // Başlangıçta skor 0
    oyuncuSkoru = 10; // Canavar yenildi, skor artık 10 oldu!
  • const : Kilitli Kasalar (Sabitler)
    Constant (Sabit) kelimesinden gelir. İçine bir değer koyduğunuzda bir daha asla değiştiremezsiniz. TC Kimlik Numaranız veya Doğum Tarihiniz gibi!
    const dogumYili = 2008;
    // dogumYili = 2009; // BUNA İZİN VERMEZ! HATA VERİR!

4. Kutuların İçine Ne Koyabiliriz? (Veri Tipleri)

Oluşturduğumuz bu kutuların içine farklı türlerde veriler koyabiliriz. En temel 3 veri tipi şunlardır:

// 1. String (Metin Verileri): Çift veya tek tırnak içinde yazılırlar.
let ogrenciAdi = "Ahmet";
let favoriDers = 'Matematik';

// 2. Number (Sayısal Veriler): Tırnak KULLANILMAZ! Sadece rakam.
let yas = 16;
let sicaklik = 25.5; // Ondalıklı sayılar nokta ile yazılır

// 3. Boolean (Mantıksal Veriler): Sadece "true" (doğru) veya "false" (yanlış) olabilir.
let liseMezunuMu = false;
let oyunBittiMi = true;
Altın Kural: Eğer bir sayıyı tırnak içine alırsanız ("16" gibi), bilgisayar onu sayı olarak değil, bir "metin" (String) olarak algılar ve onunla matematiksel toplama işlemi yapamazsınız!
