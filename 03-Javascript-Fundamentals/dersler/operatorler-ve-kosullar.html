Matematiksel Operatörler ve If-Else (Karar) Yapıları

1. JavaScript'in Hesap Makinesi: Operatörler

Bilgisayarlar temelde devasa ve çok hızlı hesap makineleridir. JavaScript'te sayısal değerler (Number) üzerinde işlem yapmak için Aritmetik Operatörleri kullanırız.

Operatör Ne İşe Yarar? Örnek
+ Toplama 5 + 3 = 8
- Çıkarma 10 - 2 = 8
* Çarpma 4 * 2 = 8
/ Bölme 16 / 2 = 8
% Mod Alma (Bölümünden Kalan) 10 % 3 = 1 (10'u 3'e bölersek kalan 1'dir)
İpucu: Mod (%) Nerede Kullanılır?
Bir sayının tek mi yoksa çift mi olduğunu bulmak için harikadır! Eğer bir sayının 2'ye bölümünden kalan (Mod 2) 0 ise o sayı çifttir, 1 ise tektir.

2. Karşılaştırma Zamanı: Kim Daha Büyük?

Karar verebilmek için önce durumları karşılaştırmamız gerekir. Karşılaştırma operatörleri bize her zaman true (Doğru) veya false (Yanlış) cevabını verir.

  • > (Büyüktür) ve < (Küçüktür)
  • >= (Büyük Eşittir) ve <= (Küçük Eşittir)
  • == (Eşittir) ve != (Eşit Değildir)
Dikkat! Eşittir (==) ve Sıkı Eşittir (===) Farkı
JavaScript'te == sadece değerlere bakar. Yani 5 == "5" derseniz, biri sayı biri metin olsa bile size true der.
Ancak === (3 eşittir) hem değere hem de veri tipine bakar. 5 === "5" derseniz, tipleri farklı olduğu için false der. Güvenli kod yazmak için her zaman === kullanmalıyız!

3. Mantıksal Operatörler (Ve / Veya)

Bazen birden fazla şartın aynı anda sağlanmasını isteriz. Burada devreye mantıksal operatörler girer.

Kafamızda Canlandıralım: Sınıf Geçme Kuralı
  • && (VE - AND): İki şart da doğru olmak ZORUNDA. "Sınıfı geçmek için ortalaman 50'den büyük olmalı VE devamsızlığın 10 günden az olmalı." İkisinden biri bozulursa sınıfta kalırsın (false).
  • || (VEYA - OR): İki şarttan BİRİ doğru olsa yeter. "Hafta sonu sinemaya gideceğiz, hava güneşli olursa VEYA yağmur yağmazsa." İki durumdan biri kurtarıyorsa sonuç başarılıdır (true).

4. Programlamanın Kalbi: If - Else Yapıları

İşte bilgisayara zeka kattığımız an! If (Eğer) ve Else (Değilse) komutları, programımızın yol ayrımlarında hangi yoldan gideceğine karar vermesini sağlar.

[Image of programming if else flowchart diagram]

Diyelim ki bir kullanıcının ehliyet alıp alamayacağını kontrol eden bir program yazıyoruz. Kuralımız: Yaş 18'den büyük veya eşitse alabilir, değilse alamaz.

let yas = 16;

// EĞER yaş 18'den büyük veya eşitse bu bloğu çalıştır
if (yas >= 18) {
    console.log("Tebrikler, ehliyet alabilirsiniz! 🚗");
} 
// DEĞİLSE bu bloğu çalıştır
else {
    console.log("Maalesef ehliyet alamazsınız. Biraz daha büyümeniz lazım. 🚶‍♂️");
}

5. Birden Fazla Seçenek: Else If

Hayat her zaman siyah veya beyaz (If/Else) değildir. Bazen aradaki gri tonlara (ikiden fazla seçeneğe) ihtiyaç duyarız. Sınav notuna göre harf notu hesaplayan bir sistem düşünelim:

let sinavNotu = 75;

if (sinavNotu >= 90) {
    console.log("Harika! Notunuz: A 🥇");
} 
else if (sinavNotu >= 70) {
    console.log("Tebrikler! Notunuz: B 🥈");
} 
else if (sinavNotu >= 50) {
    console.log("Geçtiniz! Notunuz: C 🥉");
} 
else {
    console.log("Kaldınız! Notunuz: F 💥");
}

Not: Kodlar yukarıdan aşağı doğru okunur. Hangi koşul (şart) ilk olarak doğru çıkarsa, bilgisayar o aralığa girer ve diğerlerini okumadan atlar.
